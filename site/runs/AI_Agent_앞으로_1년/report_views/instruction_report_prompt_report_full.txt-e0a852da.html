목표/범위: “AI Agent 최신 동향과 앞으로 1년 내 대비해야 할 사항”을 기술 리더/연구자 관점에서 학술 리뷰 스타일로 정리하라. 초점은 LLM agent orchestration(오케스트레이션/플로우), tool use(툴사용 설계), reliability(프로덕션 신뢰성), evals(평가/벤치마킹), security(보안/거버넌스)이며, ‘사실’과 ‘해석/권고’를 명확히 구분하라.  

필수 섹션 구성: Executive Summary, Scope & Methodology, Key Findings, Trends & Implications, Risks & Gaps, Critics, Appendix. Executive Summary는 5–7문장으로 핵심 주장→근거(출처 유형 표기)→의미/의사결정 함의 순으로 작성. Key Findings는 3–6개 항목으로 각 항목에 (1) 핵심 주장 (2) 근거(연구/웹) (3) 한계/전제(공개정보 한계 포함)를 붙여라. Trends & Implications는 관측된 신호(평가 표준화, AgentOps/관측, 워크플로 평가, 멀티에이전트 운영, 정책/리스크)에서 향후 12개월의 제품/조직/인프라 의사결정(예: 평가체계, 가드레일, 감사추적, 블라스트 레디어스, HITL 운영)을 연결하라.  

방법론/소스 정책: 제공된 아카이브(OpenAlex works 메타, Tavily 웹 검색 요약, source_index/triage)를 기반으로 “소스 삼각측량(논문/프리프린트 vs 벤더·블로그 vs 가이드)”을 수행하라. 인용은 (저자/기관, 연도, 제목, DOI/URL) 형식으로 최소 단위 근거를 제시하고, 웹 기반 근거는 [Web]로 라벨링하라. OpenAlex는 PDF/전문 추출이 없으므로 초록/메타 기반 주장에는 ‘공개정보 한계’를 명시하고 과도한 단정 금지. 보안(Security)은 1차 근거가 상대적으로 약하므로, 직접 증거 부족 영역은 Risks & Gaps에 별도 표기하고 정책/허위정보 논문은 “간접 근거”로 취급하라.  

Critics: 주류 주장에 대한 반론(예: 에이전트 과대평가, 벤더 편향, 평가의 재현성 문제, 운영 리스크 과소추정)을 헤드라인+단락+불릿으로 제시. 해당 없음이면 Not applicable과 사유를 명시. Appendix에는 사용 소스 목록(우선 12개)과 분류(연구/웹/가이드), 그리고 ‘증거 강도’ 메모를 포함하라.
